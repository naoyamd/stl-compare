# STL Compare

ローカルフォルダー内のSTLファイルを、1画面で最大200件まで並べて比較するブラウザビューアーです。すべてのモデルに同じ回転・移動・ズームを適用します。

![通信不要](https://img.shields.io/badge/network-none-56e0bd) ![依存関係なし](https://img.shields.io/badge/dependencies-0-56e0bd) ![license](https://img.shields.io/badge/license-MIT-56e0bd)

## 使い方

1. `stl_viewer.html` をChromium系ブラウザで開きます。
2. 「フォルダーを選択」からSTLファイルを含むローカルフォルダーを指定します。
3. 「同時表示」スライダーで表示件数を選びます。初期値は50件です。
4. ビュー内をドラッグして回転、`Shift`＋ドラッグまたは右ドラッグで移動、ホイールで拡大縮小します。

インストール、Webサーバー、インターネット接続は不要です。ブラウザの安全仕様上、ページへフォルダーのパスを直接入力することはできないため、OSのフォルダー選択画面を使用します。

## 機能

- Binary STL / ASCII STLの自動判別
- 1つのWebGL2コンテキストを分割した高効率な同時描画
- 最大200件、初期値50件の表示件数スライダー
- 回転・平行移動・ズーム・標準ビュー・エッジ表示の同期
- 個別フィットと共通スケールの切り替え
- 複数Web Workerによる並列STL解析
- ファイル名の自然順ソート
- 読み込んだデータをネットワークへ送らない完全ローカル処理
- ドラッグ＆ドロップによるファイル読み込み

## 設計

50個の`canvas`を作るとブラウザのWebGLコンテキスト上限に達しやすいため、単一の`canvas`を`viewport`と`scissor`でタイル分割しています。モデルごとにGPUバッファは保持しますが、シェーダー、描画状態、カメラは共有します。また、画面外タイルはスクロール時に必要となる範囲だけ再描画します。

STL解析は最大4本のBlob Web Workerで行い、解析済み頂点バッファをTransferableとしてメインスレッドへ渡します。外部ライブラリ、CDN、API、`fetch`は使用していません。

## 動作環境

- WebGL2とWeb Workerを有効にした最新のChromium系ブラウザ
- 50件以上を扱う場合は、ハードウェアアクセラレーションが有効なCAD用途相当のPCを推奨

入力データ量、三角形数、画面解像度に応じてGPUメモリ使用量は増えます。大量データでは「同時表示」件数を下げて調整してください。

## ファイル構成

- `stl_viewer.html` — UIとスタイル
- `app.js` — STL解析、WebGL2描画、同期操作
